# Browser QA

Use this skill after frontend changes or when a user asks to inspect a local page.

## Checks

- Open the relevant route in a real browser when a local server is available.
- Capture desktop and mobile screenshots for visual changes.
- Check console errors, failed network requests, and obvious accessibility regressions.
- Exercise the main interaction path, not only page load.
- Report concrete issues with route, viewport, and reproduction steps.
