# Design Brainstorming

Use this before committing to a visual direction or when a design feels generic.

## Workflow

1. Understand audience, domain, workflow, brand constraints, density, and tone.
2. Generate three to five distinct directions, not color variants.
3. For each: layout, typography, palette, motion, interaction density, and risk.
4. Recommend one direction and explain why.
5. If coding is requested, translate it into tokens, components, states, and checks.

## Guardrails

Avoid generic SaaS defaults, one-note palettes, decorative blobs, cards-inside-cards, and marketing layouts for operational tools. Keep mobile, responsive text fit, a11y, loading, empty, and error states in scope.

## Output

Return named directions, recommendation, and implementation checklist when useful.
