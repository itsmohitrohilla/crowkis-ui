# UI UX Audit

Use this skill before or after UI changes when the question is whether the product feels clear, credible, and usable.

## Audit Order

1. Task clarity: can the intended user complete the main workflow without explanation?
2. Information hierarchy: are primary actions and status signals obvious?
3. Accessibility: contrast, labels, focus, keyboard, reduced motion, and touch target size.
4. Responsiveness: no horizontal overflow, clipped controls, text overlap, or layout jumps.
5. Visual quality: consistent spacing, type scale, component density, icon style, and color use.

Report concrete fixes first. Avoid broad taste commentary unless it changes implementation choices.
