# Dashboard Prototype

Use this skill for operational interfaces that users scan repeatedly.

## Rules

- Prioritize dense but calm information architecture.
- Use sidebar or top navigation only when it helps repeated workflows.
- Show realistic tables, filters, statuses, charts, and empty/loading/error states.
- Avoid marketing hero layouts, decorative card stacks, and oversized display type.
- Keep repeated cards at 8px radius or less unless the existing design system says otherwise.
