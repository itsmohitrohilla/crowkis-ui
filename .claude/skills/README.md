# Crowkis UI/UX skill toolkit

A curated set of **20 UI/UX-design skills** for building and polishing the
Crowkis site & app. Invoke one per task with the Skill tool (e.g. `web-design`,
`ui-ux-audit`). Pick the skill that matches the job:

## Design & ideation
- **design-brainstorming** — explore visual directions before building.
- **wireframe-sketch** — lo-fi layouts / early structure.
- **frontend-design-core** — visual hierarchy, taste, non-generic polish.
- **ui-ux-pro-max** — searchable design-pattern database (67+ styles).
- **uiux-design** / **web-design** — general UI/UX + web design playbooks.

## Building surfaces
- **web-prototype** — landing / SaaS / docs / product pages.
- **dashboard-prototype** — admin, analytics, KPI dashboards (the /app preview).
- **editorial-prototype** — text-forward / docs / blog surfaces (the Roost).
- **mobile-screen** — phone-sized screens & flows.
- **frontend-component-craft** — React components, forms, registries.
- **frontend-ui-engineering** — production UI architecture, responsive, a11y states.
- **react-next-modern** — React/Next component & hook patterns.

## Components & motion
- **shadcn-registry** — shadcn/ui components & registries.
- **react-bits** — animated cards, text effects, subtle backgrounds.
- **magic-ui-motion** — Magic UI animated sections & microinteractions.
- **framer-motion** — transitions, gestures, microinteractions.

## Review & QA
- **ui-ux-audit** — review an interface for UX issues & polish.
- **live-site-visual-audit** — compare against a reference / production.
- **frontend-accessibility-review** — keyboard, screen-reader, WCAG, focus.

> Rule of thumb: one skill per task. Reach for an audit skill after building,
> a prototype skill when creating a new surface, and a motion skill for polish.
