# Frontend Design Core

Use this skill for landing pages, SaaS screens, dashboards, product pages, and app views where the output must feel production-grade.

## Working Rules

- Start from the product purpose and user workflow, not decorative composition.
- Prefer restrained, readable UI with strong spacing, clear hierarchy, and accessible contrast.
- Avoid generic AI defaults: purple gradients, oversized rounded cards, vague feature grids, fake metrics, and decorative clutter.
- Reuse the project design system and component library before inventing a new one.
- Use real assets or purposeful generated assets when visuals are central to the task.
- Verify responsive behavior before calling the UI finished.

## Output Standard

Deliver working UI code, not a marketing description. Include controls and states a real user would expect.
